## 1. データの基本操作（CRUD）

データベースのテーブルに対してデータを「追加（Create）」「更新（Update）」「削除（Delete）」する最も基本的な操作です。ここでは、**SNSの投稿を管理する `tweets` テーブル**を例に解説します。

### 📥 データの追加 (`INSERT INTO`)

テーブルに新しいレコード（行）を1件追加します。

#### 💡 構文

```sql
INSERT INTO テーブル名 (カラム名1, カラム名2, ...) 
VALUES (値1, 値2, ...);
```

#### 📝 具体例（ツイートの新規投稿）

```sql
INSERT INTO tweets (id, username, content, created_at, updated_at)
VALUES (1, 'タカシ', '初ツイート！', '2026/06/28', '2026/06/28');
```

#### 📌 補足（カラム指定の省略）

カラム名の指定を省略して `INSERT INTO tweets VALUES (1, 'タカシ', ...)` と書くことも可能ですが、その場合はテーブルに定義されているすべてのカラムに対して、順番通りに値を指定する必要があります。実務ではミスを防ぐため、カラム名を明記することが推奨されます。

### 🔄 データの更新 (`UPDATE`)

既にテーブルに存在するデータを書き換えます。

#### 💡 構文

```sql
UPDATE テーブル名
   SET カラム名1 = 値1, カラム名2 = 値2
 WHERE 条件;
```

#### 📝 具体例（特定のツイート内容の編集）

```sql
UPDATE tweets
   SET content = 'ツイート内容を修正しました！', updated_at = '2026/06/28'
 WHERE id = 1;
```

#### ⚠️ 危険：WHERE句の重要性

`WHERE` 句（条件）を書き忘れると、テーブル内にあるすべてのレコードが更新されてしまいます。更新を行う際は、必ず「どの行をターゲットにするか」を `WHERE` で指定してください。

### 🗑️ データの削除 (`DELETE`)

不要になったレコードを削除します。

#### 💡 構文

```sql
DELETE FROM テーブル名
 WHERE 条件;
```

#### 📝 具体例（特定のツイートの削除）

```sql
DELETE FROM tweets
 WHERE id = 1;
```

#### ⚠️ 危険：全削除に注意

`UPDATE` と同様に、`WHERE` 句を忘れるとテーブル内の全データが削除されて空っぽになります。実行前には必ず条件式に間違いがないか確認しましょう。

## 2. 条件指定と便利な演算子

データを検索（`SELECT`）する際、より柔軟にデータを絞り込むための演算子です。ここからは、親しみやすい「家計簿（kakeibo）」テーブルを例に進めます。

### 🔍 NULLの判定 (`IS NULL`)

データが「何も入っていない空の状態」を **NULL（ヌル）** と呼びます。NULLを検索するときは、`=` ではなく `IS NULL` を使います。

#### 📝 具体例（メモが未入力の履歴を検索）

```sql
SELECT *
  FROM kakeibo
 WHERE memo IS NULL;
```

#### 💡 データの有無を逆に判定したい場合

値が「入っている」データだけを抽出したい場合は、`IS NOT NULL` を使用します。

```sql
WHERE memo IS NOT NULL;
```

### 📏 範囲の指定 (`BETWEEN`)

「〇〇以上、△△以下」という連続した数値や日付の範囲を指定してデータを絞り込みます。

#### 💡 構文

```sql
SELECT カラム名 FROM テーブル名 WHERE カラム名 BETWEEN 下限値 AND 上限値;
```

#### 📝 具体例（出金額が1,000円〜30,000円のものを検索）

```sql
SELECT *
  FROM kakeibo
 WHERE expense_amount BETWEEN 1000 AND 30000;
```

`expense_amount >= 1000 AND expense_amount <= 30000` と書くのと同じ意味ですが、`BETWEEN` を使うことでシンプルかつ読みやすいコードになります。

### 👥 複数値の比較 (`ANY`)

指定した複数の値（または別のクエリの結果）の「いずれか」と比較して条件を満たすものを抽出します。

#### 💡 構文

```sql
SELECT カラム名 FROM テーブル名 WHERE カラム名 <= ANY (値1, 値2, 値3, ...);
```

#### 📝 具体例（指定した金額リストの「いずれかの値以下」である出金を検索）

```sql
SELECT *
  FROM kakeibo
 WHERE expense_amount <= ANY (1000, 5000, 10000);
```

この例では、`ANY` の中の一番大きい値（10000）以下であるデータ（10000円以下すべて）がヒットします。

なお、`カラム名 = ANY (値1, 値2, ...)` は、よく使われる `IN (値1, 値2, ...)` 演算子と全く同じ挙動（いずれかに一致する）になります。

## 3. データの並び替えと集計・グループ化

大量のデータから合計を算出したり、特定の項目ごとに分類したりする、データ分析・レポート作成の基本機能です。

### 📊 データの並び替え (`ORDER BY`)

取得したデータを特定のカラムを基準に並び替えて表示します。

- **ASC**: 昇順（小さい順、古い順） ※省略した場合はデフォルトで昇順になります
- **DESC**: 降順（大きい順、新しい順）

#### 📝 具体例（金額が高い順に並び替える）

```sql
SELECT *
  FROM kakeibo
 ORDER BY expense_amount DESC;
```

### 🧮 データの集計 (`SUM`, `COUNT`)

テーブル内の数値の合計やデータの件数を計算します。これらは**集計関数**と呼ばれます。

#### 📝 具体例1：合計金額の計算 (`SUM`)

「出金額」カラムの合計を計算し、結果の列名に見やすい別名（エイリアス：`AS`）を付与します。

```sql
SELECT SUM(expense_amount) AS 合計出金額
  FROM kakeibo;
```

#### 📝 具体例2：取引件数のカウント (`COUNT`)

データが全部で何件（何行）存在するかを数えます。`*`（アスタリスク）を指定すると、行数そのものをカウントします。

```sql
SELECT COUNT(*) AS 合計取引数
  FROM kakeibo;
```

### 🗂️ データのグループ化 (`GROUP BY`)

データを特定の項目ごとに「グループ」としてまとめ、そのグループごとに集計関数（`SUM`や`COUNT`など）を適用します。

#### 📝 具体例（費目ごとにグループ化し、それぞれの合計金額を算出する）

```sql
SELECT category_name, SUM(expense_amount) AS 費目別の出金合計額
  FROM kakeibo
 GROUP BY category_name;
```

#### 📋 出力イメージ

| category_name | 費目別の出金合計額 |
|---|---|
| 食費 | 25,000 |
| 交際費 | 12,000 |
| 交通費 | 5,500 |

## 4. 複数テーブルの連携

実際のデータベース設計では、重複を避けて効率よくデータを管理するためにテーブルが役割ごとに小分けにされています。それらを実務で活用するための高度な連携テクニックです。

### 🤝 テーブルの結合 (`INNER JOIN`)

2つの異なるテーブルを、共通のキー（IDなど）を基準にしてドッキングさせ、**両方のテーブルに一致するデータだけ**を1つの大きな表のように扱います。`INNER JOIN` は最も基本的な結合方法で、`JOIN` と書いた場合も同じ意味になります。

#### 💡 構文

`INNER JOIN` は、次の順番で記述します。

```sql
SELECT 列名
  FROM テーブル1
 INNER JOIN テーブル2
    ON テーブル1.キーとなる列名 = テーブル2.キーとなる列名;
```

- `FROM テーブル1` … 結合の起点となるテーブル（左側）
- `INNER JOIN テーブル2` … 結合する相手のテーブル（右側）
- `ON` … 両テーブルを結びつける条件（通常は ID などのキー列同士を `=` で結ぶ）

#### 📝 具体例（「家計簿」に「費目マスター」の名称を結合する）

`kakeibo` テーブルにある `category_id`（数字のID）をキーにして、`categories` テーブルから実際の `category_name`（食費、交通費などの名前）を紐付けて取得します。`category_id` が一致する行だけが結果に含まれます。

```sql
SELECT kakeibo.date AS 日付,
       categories.category_name AS 費目,
       kakeibo.memo AS メモ
  FROM kakeibo
 INNER JOIN categories
    ON kakeibo.category_id = categories.id;
```

### 🕸️ サブクエリ（副問合せ）

SQL文の中に、別のSQL文（子クエリ）を入れ子にして埋め込むテクニックです。「まず裏で特定の値を計算させ、その結果を使って本番の検索を行う」という2段構えの処理が可能です。

#### 📝 具体例（「平均出金額」よりも高い取引だけを抽出する）

通常、「平均値がいくらか」が事前に分かっていないと条件指定できませんが、サブクエリを使えば1回の実行で完結します。

```sql
SELECT *
  FROM kakeibo
 WHERE expense_amount > (
       /* 【サブクエリ】まず全体の平均出金額を自動計算している */
       SELECT AVG(expense_amount) FROM kakeibo
 );
```

#### 🔄 サブクエリの実行ステップ

1. カッコ内の `SELECT AVG(expense_amount) FROM kakeibo` が最初に実行され、データベースの全データから平均値（例: 3500）が導き出されます。
2. その結果が外側のメインクエリに渡され、最終的に `WHERE expense_amount > 3500` という条件でデータが高速に検索されます。
